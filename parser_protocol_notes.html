1 Introduction

The burrito subprocess protocol is a subclass of the original AsyncIO SubprocessProtocol class, that passes the data arriving on the input pipe file descriptors to chunk parsers with a regular expression set relative to the file descriptor no. In other terms, there is a dictionary for each process output number, containing the regexs to match in the stream for the output number (and a handler for the matches, that can be None).

In a Burrito, this subprocess protocol is used to communicate with the process and parse its standard error and output with the given matching regexes.

1.1 Subprocess execution

The subprocess protocol can provide a coroutine function that executes the process. This can be used to automatically set stdin to None, which is needed to avoid blocking on a process waiting for standard input when not needed (as it's the case with Micmac). This coroutine function would need to be a classmethod, as we can not instantiate the protocol directly, it is instantiated when running the coroutine (when executing the function in AsyncIO to execute a process, we give it a "protocol factory", a callable that returns a protocol instance that can be used (with the AsyncIO transport) to communicate with subprocess

1.2 Subprocess protocol signature

The subprocess protocol constructor takes 3 keyword arguments, that can be used as positional arguments in the following order :

started
(default value None) Future that completes when the process starts (if None, will be automatically created)
exited
(default value None) Future that completes when the process exits (if None, will be automatically created)
loop
(default value None) AsyncIO event loop (if None, will be automatically initialized using burrito.aio.loop.get_event_loop)

NOTE: burrito.aio.loop.get_event_loop is needed because on Windows ("NT" platforms), we have to set the event loop to a ProactorEventLoop to use pipes on processes. Because of this, loop needs to pass burrito.aio.loop.check_event_loop when running in non-optimized mode (by using an assertion).

1.3 make_parser abstract method

The subprocess protocol is an abstract class, and an implementation must be provided for its make_parser method. This method will be called when data arrives on a file descriptor and no parser is available for this file descriptor.

NOTE : This is implemented, for example, by the SubprocessChunkParserProtocol subclass, that uses a chunk parser (burrito.chunk_parser.ChunkParser) to match regular expressions on the data stream.
